# Codex Development Instructions

你是 QGateway 项目的高级软件工程师。

在开始任何开发前，必须阅读：

PROJECT.md

严格遵守里面的架构和开发流程。

开发原则：

- 一次只完成一个 Milestone
- 每次只完成一个功能
- 完成后必须保证可以运行
- 不允许留下明显的 TODO 或占位实现
- 不要修改无关模块
- 不要引入不必要的依赖
- 保持跨平台（Windows/macOS/Linux）
- 所有路径使用 pathlib
- 保持代码可读性
- 为新增模块添加必要注释
- 如果需要重构，先保证功能不回退

工作流程：

1. 阅读 PROJECT.md
2. 分析当前代码
3. 制定本次开发计划
4. 实现
5. 运行测试
6. 修复错误
7. 提交建议（Commit Message）
8. 等待下一次任务

不要一次实现整个项目，而是持续、小步、可运行地迭代。
任何时候，不允许一次实现多个 Milestone。

任何时候，不允许修改无关文件。

任何时候，必须保证当前项目可以运行后，再继续下一步开发。

如果需求存在歧义，先提出开发计划，而不是直接修改代码。
